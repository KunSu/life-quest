# Milestone 0.5 Contract: Dual Theme System

## API Interface

- `PATCH /api/user/theme`
- Request type: `{ themeMode: 'modern' | 'adventure' }`
  - Response type: `{ data: { themeMode: 'modern' | 'adventure' } }`
  - Error type: `{ error: { code: string, message: string, status: number } }`
  - Side effects:
    - Updates `users.theme_mode` for the seeded demo user

## DB Schema Diff

- No new table in Milestone 0.5
- Reuse existing `users.theme_mode TEXT NOT NULL`

## Page Routes

- `/dashboard` - renders the current SSR theme on `<html data-theme>`

## Zod Schemas

- `src/shared/schemas/user.ts`
  - `updateThemeSchema = z.object({ themeMode: z.enum(['modern', 'adventure']) })`

## Acceptance Criteria

- Given the app loads with no cookie and a `modern` DB value, when I open `/dashboard`, then `<html data-theme="modern">` is present on first render
- Given I switch to `adventure`, when the mutation succeeds, then the response sets the theme cookie and the demo user row is updated to `adventure`
- Given I refresh the page after switching theme, when `/dashboard` reloads, then `adventure` remains active
- Given I open a fresh browser context after the mutation, when no cookie is present, then the server still resolves `adventure` from SQLite
- Given the dashboard is shown on mobile, when I use the theme switcher, then no horizontal overflow or layout break appears

## What You Should Review

### Product Review

- Open `/dashboard`
- Confirm the app can switch between `modern` and `adventure`
- Confirm the theme switcher feels discoverable on mobile
- Confirm the chosen theme survives refresh

### UI/UX Review

- Confirm `modern` feels like a growth-product skin, not a placeholder
- Confirm `adventure` feels clearly different without changing layout structure
- Confirm there is no visible flash from the wrong theme during initial load

### Behavior Review

- `PATCH /api/user/theme` only accepts `modern` or `adventure`
- The route writes the new theme to SQLite and sets the theme cookie
- `app/layout.tsx` resolves the initial theme before rendering `<body>`

### Out of Scope For M0.5 Review

- Quest CRUD
- Task CRUD
- Reward animations
- Dashboard data widgets beyond the shell
